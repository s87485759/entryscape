General principles

In this folder (the Confolio-install/lib) all external libraries should be placed. Each library X may end up in three distinct places, for convenience we normalize the naming and describe what we expect of them:
lib/X-src/ the original library
lib/X/ a version of the library suitable for running and debugging
build/release/X/ a compressed/built version of the library suitable for a production evironment, only libraries built using the dojo package mechanism will end up here

Libraries upon which Confolio depends

dojo - The Dojo tookit

We currently rely on version 1.2.3 of the Dojo toolkit. You can download from:
http://download.dojotoolkit.org/release-1.2.3/
As the releases since 1.2 does not include debugging variants (not compressed with Shrinksafe) the stock release is in itself not sufficient. Instead we recommend that you download the source (dojo-release-1.2.3-src.tar.gz) and unpack to dojo-src/ and then produce the debugging environment via the buildscripts. To produce the running/debugging version:

$ cd dojo-src/util/buildscripts/
$ build.sh profile=standard action=release releaseDir=../../../dojo/
You can also run the script INSTALL-dojo.sh, it will download and build everything for you.

SHAME - A library for metadata authoring, the javascript part

See the homepage for more information on how SHAME works.
Currently there is a few deviations from the regular SHAME release, hence it is provided in SVN directly as 'shame-src'.
